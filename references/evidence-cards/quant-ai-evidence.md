# Quant AI Evidence

## Use When

Use for fixed-income quant research, AI research agents, factor design, monitoring systems, report automation, and data-contract governance.

## Source Reports

- `huatai-quant-ai-ficc-2025`
- `cicc-ficc-ai-agent-path-2025`

## Core Framework Claims

- AI research workflows should start from a defined research question, data contract, validation design, and human-review boundary.
- LLMs are useful for parsing, routing, summarization, candidate generation, and monitoring, but are not themselves verified market-data sources.
- Fixed-income factors require point-in-time data, sample-out validation, cost awareness, and failure-mode logging.

## Data Fields Mentioned

- Data dictionary, source, timestamp, frequency, universe, labels, factor definitions.
- Return, spread changes, drawdowns, redemption events, default or extension events, policy events.
- Backtest metrics, turnover, transaction cost, out-of-sample period, alert thresholds.

## Common Misreadings

- Treating an LLM summary as a data fact.
- Accepting a factor without out-of-sample validation.
- Ignoring point-in-time availability or look-ahead bias.
- Letting automated reports become investment decisions without review.
